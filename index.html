Clean Architecture Artifacts

There are many artifacts in Clean Architecture. It is important to understand their role and why they are there.

Controller

The Controller takes the HTTP Request and transforms it into the UseCase Request.

UseCase Request

The UseCase Request is a data structure without any logic. It contains request data as a plain DTO (Data Transfer Object).

UseCase Interactor

The Interactor orchestrates the use case. It takes the data from the request, validates it, fetches the Entities from the Repository Interface, let the Entity execute the business rules, and save the Entities in their new states via the Repository Interface.

Repository Interface

Defines the interface on how to access and save data to the Data Source.

Entities

The Business Objects. Contain all the Business Rules. When you like to do Domain Driven Design, this is the Domain Model.
